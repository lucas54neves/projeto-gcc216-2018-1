Boas práticas para o código:
1) Variáveis com letra minúscula;
2) Todo atributo de classe começa “m” minúscula ou “_”;
3) Constantes com todas as letras maiúsculas;
4) Usar nomes de funções e operações da classe com letra maiúscula;
5) Se a variável não usar número negativos usar “unsigned int”;
6) Para comparação entre os tipos “unsigned int” e “int”, usar “long”;
7) Símbolo do ponteiro ao lado do tipo(Ex.: int* ptr);
8) Usar nomes práticos (óbvios) para as funções, de acordo com o que elas retornam;
9) Para deletar vetores, colocar “delete[] vetor;”;
10) Para operações matemáticas, usar seguinte exemplo “a > b”;
11) Colocar comentário em cada módulo e em situações extras que o programador achar necessário;
12) Dar uma linha de espaço entre cada módulo;
13) Separar tipos de operações com espaço dentro de cada subprograma;
14) Nomes de classes e estruturas sempre começam inicial maiúscula;
15) Usar palavras em português;
16) Para nomes compostos, a primeira palavra tem a inicial minúscula e as palavras seguintes tem iniciais maiúsculas.
